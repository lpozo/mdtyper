## Architecture

```
VSCode Extension Host (TypeScript / Node.js)
  тФФтФАтФА MdTyperEditorProvider  (CustomTextEditorProvider)
        тФФтФАтФА WebviewPanel
              тФФтФАтФА Milkdown editor  (ProseMirror + Remark, browser)
```

**Key libraries:**
- **Milkdown** (`@milkdown/kit`) тАФ ProseMirror + Remark, lossless Markdown round-trip
- **esbuild** тАФ dual-target build: Node.js CJS for extension host, browser IIFE for webview

**Key VS Code APIs:**
- `CustomTextEditorProvider` тАФ routes `.md` files to MdTyper; VS Code handles save/dirty/undo
- `WebviewPanel` тАФ hosts the Milkdown editor as HTML/CSS/JS
- `retainContextWhenHidden: true` тАФ keeps Milkdown alive across tab switches

## Source Layout

## Message Protocol

| Direction | Message | Purpose |
|-----------|---------|---------|
| Webview тЖТ Host | `{ type: 'ready' }` | Webview loaded; request initial content |
| Host тЖТ Webview | `{ type: 'update', content: string }` | Send/refresh document content |
| Webview тЖТ Host | `{ type: 'edit', content: string }` | User made a change |
| Webview тЖТ Host | `{ type: 'link', href: string }` | User clicked a link; host resolves and opens it |

**Echo-loop guards:** `isUpdatingFromWebview` (`MdTyperEditorProvider.ts`) and `isUpdatingFromHost` (`main.ts`) тАФ both must be reset in `finally`.

## TypeScript & Code Quality

### Compiler strictness
Both `tsconfig.json` and `tsconfig.webview.json` enable:

| Flag | Why |
|------|-----|
| `strict: true` | Enables `strictNullChecks`, `noImplicitAny`, and related checks |
| `noUncheckedIndexedAccess: true` | Array/bracket access includes `\| undefined` тАФ forces explicit null checks |
| `exactOptionalPropertyTypes: true` | Distinguishes `prop?: T` from `prop: T \| undefined` тАФ protects message shapes |

> `tsconfig.webview.json` is **type-check only** тАФ esbuild reads `webview-src/` directly and emits to `dist/`. The `outDir` in that tsconfig is never written to.

### ESLint rules (beyond recommended)
- `no-unused-vars` тАФ params/vars prefixed `_` are intentionally unused; all others are errors
- `no-non-null-assertion` тАФ `!` assertions are warnings; use explicit null checks
- `no-explicit-any` тАФ `any` is an error; use `unknown` and narrow before use

### Type safety rules
- **`event.data` is `unknown` at runtime** тАФ narrow with a type guard, never cast with `as`. See `isUpdateMessage()` in `webview-src/main.ts`.
- **Never use `!` on DOM queries** тАФ `getElementById()` can return `null`; throw a descriptive error instead. See `initEditor()` in `webview-src/main.ts`.
- **`acquireVsCodeApi()`** тАФ type is provided by `@types/vscode-webview`; do not redeclare manually.

### Error handling rules
- **`applyEdit` returns `boolean`**, not throws тАФ `false` means conflict; check the return value and log it. See `case 'edit'` in `MdTyperEditorProvider.ts`.
- **Async `addEventListener` callbacks** тАФ wrap body in `void (async () => { try { тАж } catch { тАж } })()` to prevent silent rejection. See message handler in `webview-src/main.ts`.
- **Echo-loop guard flags** тАФ always reset in `finally` so a throw cannot permanently freeze sync. Both source files follow this pattern.

## Critical Implementation Notes

- **`external: ['vscode']` in `esbuild.mjs` is mandatory** тАФ `vscode` is a virtual module injected by the extension host runtime; never bundle it.
- **`acquireVsCodeApi()` must be called exactly once** per webview lifetime тАФ stored in a module-level const in `main.ts`.
- **Use `vscode.workspace.applyEdit`** to modify the document тАФ never `fs.writeFile`, which bypasses VS Code's undo stack and dirty tracking.
- **`retainContextWhenHidden: true`** keeps Milkdown alive across tab switches at higher memory cost тАФ correct default for a WYSIWYG editor.
- **`theme.css` is a runtime asset** тАФ it lives in `webview-src/` but must be in `dist/` for the packaged extension. `esbuild.mjs` copies it; `--watch` mode does **not** auto-copy on CSS changes.
