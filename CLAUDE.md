# MdTyper ‚Äî WYSIWYG Markdown Editor for VS Code

Typora-like inline WYSIWYG Markdown editor for VS Code. TypeScript only ‚Äî the extension host runtime does not support other languages.

## Architecture

```
VSCode Extension Host (TypeScript / Node.js)
  ‚îî‚îÄ‚îÄ MdTyperEditorProvider  (CustomTextEditorProvider)
        ‚îî‚îÄ‚îÄ WebviewPanel
              ‚îî‚îÄ‚îÄ Milkdown editor  (ProseMirror + Remark, browser)
```

**Key libraries:**
- **Milkdown** (`@milkdown/kit`) ‚Äî ProseMirror + Remark, lossless Markdown round-trip
- **esbuild** ‚Äî dual-target build: Node.js CJS for extension host, browser IIFE for webview

**Key VS Code APIs:**
- `CustomTextEditorProvider` ‚Äî routes `.md` files to MdTyper; VS Code handles save/dirty/undo
- `WebviewPanel` ‚Äî hosts the Milkdown editor as HTML/CSS/JS
- `retainContextWhenHidden: true` ‚Äî keeps Milkdown alive across tab switches

## Message Protocol

| Direction | Message | Purpose |
|-----------|---------|---------|
| Webview ‚Üí Host | `{ type: 'ready' }` | Webview loaded; request initial content |
| Host ‚Üí Webview | `{ type: 'update', content: string }` | Send/refresh document content |
| Webview ‚Üí Host | `{ type: 'edit', content: string }` | User made a change |
| Webview ‚Üí Host | `{ type: 'link', href: string }` | User clicked a link; host resolves and opens it |

**Echo-loop guards:**
- `isUpdatingFromWebview` (`MdTyperEditorProvider.ts`) ‚Äî set before `applyEdit`, always reset in `finally`.
- `isUpdatingFromHost` (`webview-src/editor-service.ts`) ‚Äî set via `setHostUpdateGuard()`, auto-resets after 300 ms. Timer-based (not `finally`) because it must outlast Milkdown's internal 200 ms debounce before a synchronous reset would be possible.

## TypeScript & Code Quality

### Compiler strictness
Both `tsconfig.json` and `tsconfig.webview.json` enable:

| Flag | Why |
|------|-----|
| `strict: true` | Enables `strictNullChecks`, `noImplicitAny`, and related checks |
| `noUncheckedIndexedAccess: true` | Array/bracket access includes `\| undefined` ‚Äî forces explicit null checks |
| `exactOptionalPropertyTypes: true` | Distinguishes `prop?: T` from `prop: T \| undefined` ‚Äî protects message shapes |

> `tsconfig.webview.json` is **type-check only** ‚Äî esbuild reads `webview-src/` directly and emits to `dist/`. The `outDir` in that tsconfig is never written to.

### ESLint rules (beyond recommended)
- `no-unused-vars` ‚Äî params/vars prefixed `_` are intentionally unused; all others are errors
- `no-non-null-assertion` ‚Äî `!` assertions are warnings; use explicit null checks
- `no-explicit-any` ‚Äî `any` is an error; use `unknown` and narrow before use

### Type safety rules
- **`event.data` is `unknown` at runtime** ‚Äî narrow with a type guard, never cast with `as`. See `isUpdateMessage()` in `webview-src/main.ts`.
- **Never use `!` on DOM queries** ‚Äî `getElementById()` can return `null`; throw a descriptive error instead. See DOM element queries at the top of `webview-src/main.ts`.
- **`acquireVsCodeApi()`** ‚Äî type is provided by `@types/vscode-webview`; do not redeclare manually.

### Error handling rules
- **`applyEdit` returns `boolean`**, not throws ‚Äî `false` means conflict; check the return value and log it. See `case 'edit'` in `MdTyperEditorProvider.ts`.
- **Async `addEventListener` callbacks** ‚Äî wrap body in `void (async () => { try { ‚Ä¶ } catch { ‚Ä¶ } })()` to prevent silent rejection. See message handler in `webview-src/main.ts`.
- **Echo-loop guard flags** ‚Äî always reset in `finally` so a throw cannot permanently freeze sync. Both source files follow this pattern.

## Critical Implementation Notes

- **`external: ['vscode']` in `esbuild.mjs` is mandatory** ‚Äî `vscode` is a virtual module injected by the extension host runtime; never bundle it.
- **`acquireVsCodeApi()` must be called exactly once** per webview lifetime ‚Äî stored in a module-level const in `main.ts`.
- **Use `vscode.workspace.applyEdit`** to modify the document ‚Äî never `fs.writeFile`, which bypasses VS Code's undo stack and dirty tracking.
- **`retainContextWhenHidden: true`** keeps Milkdown alive across tab switches at higher memory cost ‚Äî correct default for a WYSIWYG editor.
- **`theme.css` is a runtime asset** ‚Äî it lives in `webview-src/` but must be in `dist/` for the packaged extension. `esbuild.mjs` copies it; `--watch` mode does **not** auto-copy on CSS changes.
